Privacy Policy

AI Scanner — Last updated: June 15, 2026

This extension helps you understand cosmetic / product ingredient lists on supported retailer sites by sending limited page-derived data to an API server you configure (for example your own backend or a host you trust). This policy describes what the extension itself collects and sends. Your API operator may apply additional practices; review their policy if you use a third-party server.

1. Who this applies to

This policy covers the Chrome extension AI Scanner. It does not replace the privacy policy of the retailer websites you visit or of the API service you point the extension at.

2. Single purpose

The extension has one primary purpose: when you choose to analyze a product page, it extracts ingredient-related information and requests an analysis from your configured API. It does not inject ads, silently modify pages for unrelated purposes, or combine unrelated features.

3. Data the extension collects and uses

When you click Analyze Product (or equivalent), the extension may collect and use:

  • Page URL of the product — to identify the product context and support caching on the server.
  • Site / retailer identifier — to choose the correct parsing strategy.
  • Product title or name — as shown on the page.
  • Ingredient-related text extracted from the visible page (for example from an ingredients section).
  • Image URLs (and optional dimensions or alt text when available) — so the API can optionally run OCR on product images when you enable that mode.
  • Optional retailer product id — when the page exposes a stable id.
  • Your settings stored in Chrome (API base URL, optional API key, analysis preferences).

The extension does not read passwords, payment fields, or unrelated form data. It does not run hidden analytics or load executable code from remote servers.

4. Where data is sent

Analysis requests are sent with HTTPS (or http:// only if you deliberately configure a local development URL such as localhost). The request goes only to the API Base URL you set in extension options, plus paths allowed by the extension manifest for that host.

Google Cloud Vision (OCR): The extension does not call Google directly. If your API is configured with Vision credentials, image-derived text processing may occur on the server side. That is governed by your API operator and Google’s terms, not by code loaded from the network inside the extension.

5. Why data is collected

  • To request ingredient parsing, matching, and tiered safety-style summaries from your API.
  • To show results in the side panel and optional on-page banner after you initiate an analysis.
  • To remember your preferences (API URL, keys, toggles) across devices if you use Chrome sync for extension settings.

6. Storage

  • chrome.storage.sync — extension settings (e.g. API URL, optional API key, feature toggles).
  • chrome.storage.local — short-lived state for the current analysis job and last result so the UI can update.

You can clear extension data from Chrome’s extension details page for this extension.

7. Permissions (why they are requested)

  • activeTab — interact with the tab you are analyzing when you take action in the extension.
  • scripting — run the content script to read public product page content you asked to analyze.
  • storage — save your settings and temporary analysis state.
  • sidePanel — show results in Chrome’s side panel.
  • tabs — resolve the active tab and send messages between the side panel, background worker, and content script for the analysis you started.
  • host_permissions — limited to supported retailer origins, your API host(s), and image CDN patterns needed to collect image URLs used for analysis (for example Amazon image hosts).

8. Remote code

The extension is built as packaged JavaScript. It does not download or execute remote JavaScript, and it does not use eval() on remote content for extension logic. Network calls are used only to POST analysis payloads to your configured API and receive JSON responses.

9. Children

The extension is not directed at children under 13.

10. Changes

We may update this policy when behavior or permissions change. The “Last updated” date at the top will change accordingly. Material changes should also be reflected in the Chrome Web Store listing when you publish updates.
